06
Core Concept

Cuộc chia tách vĩ đại: Client vs Server

Cuộc cách mạng về Separation of Concerns (Phân chia lao động).

Client (Front-end)

Hiển thị giao diện (UI)
Tương tác người dùng (UX)
Trình duyệt (Browser)

Server (Back-end)

Lưu trữ dữ liệu (DB)
Xử lý Logic nghiệp vụ
Bảo mật (Security)

07
Benefits

Tại sao việc "tách rời" lại quan trọng?

  • Server: Không cần biết Client là PC, Mobile hay Tablet.
  • Client : Không cần biết Server chạy Linux hay Windows.
  • Independent Scale: Tự do phát triển công nghệ cho cả hai phía.

13
Core Principle

Statelessness (Tính Phi trạng thái)

Không có bộ nhớ lưu trữ lịch sử của các Request

Server "quên" ngay Client là ai sau khi trả lời xong.

Tại sao?

Để giải phóng bộ nhớ. Server không thể nhớ "chuyện đời tư" của hàng triệu Client.

Tác động

Mỗi Request mới đều phải tự giới thiệu lại (gửi kèm Token).
